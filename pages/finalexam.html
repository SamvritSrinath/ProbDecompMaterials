    Game Design: Top-Down Decomposition

    In this question, we are describing a programming problem that we might want to solve, but we are not interested in the actual code. Instead, we are interested in the function decomposition of this problem.

    We’re interested in designing a text-based game where a player battles monsters.

    The player and monster each have their own values for:

    • Strength: how strong they are
    • Health: their current health level

    The general flow of the game consists of the following: the player battles a monster, visits the store, battles the next monster, visits the store again, and so on.

    The player wins the game if they defeat 10 monsters (win 10 battles). The player loses the game if their health goes to 0.

    To battle a monster, the game first decides on the strength and health of the next monster. The monster strength and health is random but based on the total number of monsters that the player has defeated (this way the monsters get more difficult as the game progresses).

    The player and monster take turns, and the battle continues until the player or monster loses (has 0 health).

    • On each of the player’s turns, they can either do:
      • a weak attack that is guaranteed to hit the monster and will do damage (reduce health of monster) based on the player’s strength.
      • or the player can do a strong attack that is less likely to hit, but will do more damage to the monster.
    • The monster then attacks (the strength of its attack is based on the monster’s strength value). The player's health is then decreased by this amount.
    • If the player loses the combat, the game ends. If the player defeats the monster, they earn money (more money for more difficult monsters), and then they return to the store where they are automatically healed, meaning that their health goes back up to full health.

    At the shop, players can buy from the merchant.

    • The merchant offers a random selection of goods that either increase player health or player strength.
    • The cost and benefits of the merchant’s goods are related to the number of monsters defeated, but with some random variation.

    Game Design: Function Parameters

    Consider the same game description from the previous question. Consider the shop() function. Which of the following options is the most appropriate parameters and types that we should pass to this function?

    1. def shop(current_strength: int, current_health: int, monsters_defeated: int)
    2. def shop(current_money: int, current_strength: int, current_health: int)
    3. def shop(current_money: int, current_strength: int, current_health: int, monsters_defeated: int)
    4. def shop(current_money: int, current_strength: int, current_health: int, monsters_defeated: int, list_of_monsters_defeated: list)

    Correct Option: C

    Reasoning:

    • The player needs money to buy items, so current_money is essential.
    • The player's current_strength and current_health are needed to display their stats and show the effects of purchased upgrades.
    • The problem states that the "cost and benefits of the merchant’s goods are related to the number of monsters defeated," so monsters_defeated is a necessary parameter to generate the shop's inventory correctly.
    • Passing a list_of_monsters_defeated is redundant, as the shop's logic only depends on the *count* of defeated monsters, not which specific ones.

    Game Design: Docstrings

    Consider the same game description from the previous questions. We want to write a docstring for our select_monster() function. Which of the following docstrings is most appropriate?

    1. "Randomly generate an ordered list of 10 monsters for the player to battle during the game."
    2. "Generate the next monster based on how many monsters the player has defeated. Return the new monster’s health."
    3. "Generate the next monster based on how many monsters the player has defeated. Return the new monster’s strength and health."
    4. "An ordered list of all monsters was randomly created when the game started. Return a random monster from that list."

    Correct Option: C

    Reasoning:

    • The docstring should accurately describe the function's purpose and its return values. The problem states that the monster's "strength and health is random but based on the total number of monsters that the player has defeated."
    • Option A is incorrect because it describes generating all monsters at once, not one at a time.
    • Option B is incomplete because the monster has both strength and health stats that need to be generated and returned for the battle.
    • Option D contradicts the game's logic, which requires monsters to get progressively more difficult based on the number defeated, not drawn randomly from a pre-generated list.

    Spell Checker Implementation

    Given a word and a dictionary, your task is to find all the correctly spelled words (from the dictionary) that are within 1 letter distance of the word.

    The valid operations include:

    • Insert: Insert a single character anywhere into the string including before or after the first character.
    • Remove: Remove any single character.
    • Replace: Replace a character anywhere in the string with another character.

    The goal is to return a list of spelling suggestions even if the word is already in the dictionary (but the word should be in that list).

    Within the file named spell_check.py, your task is to complete the code for the functions described in it. The small_wordlist.txt and wordlist.txt files contain English words, one per line. Your code will be evaluated only using the wordlist.txt file on some specific test cases.

    Examples

    Example 1:

    Input:
    English_Dictionary: ["cat", "cot", "bat", "bit", "bite", "boot", "bo", “bot”],
    Misspelled_word: “bot”

    Output: ['cot', 'bat', 'bit', 'boot', 'bo', 'bot']

    Explanation: From the word "bot", we can get "cot" (by replacing 'b' with 'c'), "bat" (by replacing 'o' with 'a'), “bit” (by replacing 'o' with 'i'), "boot" (by adding 'o' between ‘o’ and ‘t’), "bo" (by removing 't'), and since the "bot" is already present in the dictionary return that as well. The function would return 'cot', 'bat', 'bit', 'boot', 'bo', and 'bot' as a valid list of possible spelling suggestions.

    You should define the following functions:

    • create_dictionary: Input is a file of English words, one per line. Output is a dictionary of words as keys. The values are up to your chosen implementation.
    • words_insert_char: Input is a word and a dictionary. Output is a list of words with one character added that are in the dictionary.
    • words_delete_char: Input is a word and a dictionary. Output is a list of words with one character deleted that are in the dictionary.
    • words_replace_char: Input is a word and a dictionary. Output is a list of words with one character replaced that are in the dictionary.
    • spell_check: Input is a dictionary and a word to check. Output is a list of possible corrections.

    NOTE: You are expected to complete both the given functions and also any new functions that you might come up with for problem decomposition.

    1. Design Hint 1: Think about using an alphabet string (a string with all the English characters occurring once). Also, consider which words might be formed if you used each character from that string. The only characters you need to consider for this problem are the 26 lowercase letters a-z. No numbers, uppercase letters, or special characters are necessary.
    2. Hint #2: For words_delete_char Think about where you can omit or "skip" a character
    3. Hint #3: In the final spell_check function, consider how you can handle duplicate words (spelling suggestions) as your implementation may create duplicates depending on your problem decomposition. DO NOT remove duplicates in the three intermediate functions words_insert_char, words_delete_char, and words_replace_char.
