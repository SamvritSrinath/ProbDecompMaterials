    Problem Decomposition for Student Data Processing

    Scenario: Consider that you’re a researcher doing some analysis on the amount of coursework and the quality of grades. You are given a list of dictionaries, where each dictionary represents a student. Each student dictionary contains keys such as 'name' (string), 'major' (string), units (integer), and 'grades' (a nested dictionary mapping course names (string) to letter grades (string, e.g., 'A', 'B', 'C')). You need to write a program to:

    • Calculate the total units for all students.
    • Find the average units for students in a specific major (e.g., "Computer Science").
    • Identify all students who have a particular grade (‘A’ - ‘F’).
    • Determine which major has the highest average units.

    Question: Consider decomposing this problem into smaller, manageable functions. From the list below, select all the functions that would be useful and align with good decomposition principles for solving this problem.

    1. get_total_units(students_list): Takes the list of students and returns the sum of all the units collectively.
    2. calculate_major_average_units(students_list, major_name): Takes the list of students and a major name, and returns the average units for students in that major.
    3. find_students_with_grade(students_list, grade:str): Takes the list of students, and a grade (‘A’ - ‘F’) and returns a list of names of students who have that grade.
    4. calculate_total_students(student_list): Takes a student_list and returns the total number of students in the list.
    5. find_highest_average_major(students_list): Takes the list of students and returns the name of the major with the highest average units.
    6. process_all_student_data(students_list): A single function to perform all required calculations and printing.
    7. calculate_num_courses(student): Takes in a single student and returns the total number of courses that a student took.
    8. calculate_average_GPA(student) Takes in a single student and returns the average GPA Points by converting letter grades into numbers between [0-4].

    Identifying Decomposition Steps for Complex 2D List Processing

    Question: You are given a 2D list grid representing a Square Matrix of integer values. You need to write a function analyze_grid(grid) that performs three main tasks:

    • Calculate the sum of all elements in the grid.
    • Find the maximum value in each row of the grid.
    • Count the number of "encircled" elements. An "encircled" element is defined as any element where all its direct neighbors (up, down, left, and right) exist within the grid boundaries and have a value strictly greater than the element itself. Elements on the boundary or with fewer than 4 direct neighbors within the grid cannot be encircled by this definition.

    The function should return a tuple containing the total sum, a list of row maximums, and the count of encircled elements in that specific order: (total_sum, [row_maximums], encircled_count).

    Consider the principles of Problem Decomposition discussed in Chapter 7, particularly Sections 7.1, 7.4, 7.5, and 7.6. Which of the following approaches or helper functions would be useful in creating a well-decomposed solution for parts of this problem? (Select All)

    1. Write a single function analyze_grid containing nested loops to calculate the sum, find row maximums, and count encircled elements all within the same function body.
    2. Create a helper function calculate_grid_sum(grid) that iterates through the grid and returns the total sum.
    3. Create a helper function find_row_maximum(row) that takes a list of numbers (a single row) and returns the maximum value in that row. Assume the row is not empty.
    4. Create a helper function is_element_encircled(grid, r, c) that determines if the element at (r, c) meets the definition of being encircled by strictly larger neighbors.
    5. Create a helper function get_neighbor_values(grid, r, c) that returns a list of the values of the direct neighbors for the element at (r, c), handling boundary checks.
    6. Create a function that modifies the input grid in place by setting encircled elements to 0.
    7. Create a helper function find_minimum_value(grid) that finds the smallest element in the entire grid.
    8. Create a function that modifies the input grid by replacing all negative numbers with zero.
    9. Create a helper function process_single_element(grid, r, c) that performs multiple checks and calculations for a single element, including checking if it's encircled and adding its value to a running total.
    10. Create a helper function is_valid_coordinate(grid, r, c) that checks for whether or not a specified coordinate is found within the dimensions of the grid.
